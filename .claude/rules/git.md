# Git Rules

## 커밋 메시지

**Conventional Commits** 형식을 따른다.

### 형식

```
<type>: <제목>
```

- 언어: 항상 **한국어**
- 제목만 사용 (본문, 푸터 생략)
- 제목 70자 이내, 마침표 금지

### 타입

| 타입 | 설명 |
|------|------|
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `refactor` | 기능 변경 없는 코드 리팩토링 |
| `test` | 테스트 코드 추가/수정 |
| `docs` | 문서 수정 |
| `style` | 포맷, 세미콜론 등 스타일 변경 |
| `chore` | 빌드 설정, 의존성 등 기타 변경 |

### 예시

```
feat: 상품 목록 페이지네이션 추가
fix: 장바구니 수량 업데이트 버그 수정
refactor: 주문 서비스 도메인 로직 분리
test: 회원가입 유효성 검사 테스트 추가
chore: 의존성 버전 업그레이드
```
